üì¶ 2. THE BLOCK: The Secure Container

A block bundles transactions together. It needs these 3 critical items to be valid:

  • Transactions: List of unconfirmed data/payments.
  • Timestamp: When the block was created.
  • Previous Hash: The fingerprint of the *last* block, creating the "chain."

‚õèÔ∏è 3. MINING: The Proof-of-Work Race

Miners compete to solve a complex puzzle to validate and add the new block.

  • The Puzzle: Find the **Nonce** (Number used Once).
  • The Goal: Hash the block data + Nonce to get a hash that meets the network's Difficulty Target (e.g., starts with specific zeros).
  • The Reward: First miner to solve it wins currency and transaction fees.

üõ°Ô∏è 4. HOW IT SECURES THE CHAIN

Immutability (Unchangeable)

If anyone changes a transaction in Block 5, the hash of Block 5 changes. Since Block 6 relies on the old Block 5 hash, the chain breaks instantly. This makes tampering computationally impossible.

Proof-of-Work (PoW)

The massive computational effort required to find a valid hash proves that real "work" was done. To successfully attack the chain, you would need to out-mine the entire rest of the network (a 51% attack), which is too expensive.

Think of it this way: Hashing is the secure lock on the data. Mining is finding the unique key to seal the lock and link it to the next one.
